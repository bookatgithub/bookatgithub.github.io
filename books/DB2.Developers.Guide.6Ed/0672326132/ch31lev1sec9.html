REPAIR and Versions

Recall from Chapter 7, "Database Change Management and Schema Evolution," that DB2 will create new versions of DB2 objects when they are changed using the online schema evolution features of DB2 V8.


The VERSIONS parameter of the REPAIR utility can be used to update the versions of a specific table space or index in the DB2 Catalog and Directory from the version information in the table space or index. Consider using the VERSIONS parameter when you perform the following tasks when you use the OBIDXLAT option of DSN1COPY to move objects from one DB2 subsystem to another.
