Packages for Static SQL

Static SQL is supported in distsributed applications by packages. To access remote locations using SQL embedded in an application program, the program must be precompiled and then bound into a package. The application program calls the SQL API, which executes the package at the RDBMS.

If the application program requires access to multiple RDBMSs, multiple packages must be bound, one at each location. Packages enable a request originating from one location to execute static SQL at remote locations. Of course, dynamic SQL is also supported using system-directed distribution.
